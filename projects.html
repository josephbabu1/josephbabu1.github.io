Chaos-Driven Time-Varying Convolutional Coding (CD-TVCC)

IEEE-style research paper — MATLAB simulation, Verilog RTL implementation
Complete

A rate-1/2, constraint-length-7 convolutional code whose generator polynomial pair is switched at every bit epoch by a synchronized chaotic sequence, rather than fixed for the whole transmission. The generator pool is restricted to four pre-selected K=7 polynomial pairs that all meet a minimum free distance of 10, so switching adds physical-layer obfuscation without costing any error-correction performance. A receiver that shares the same chaotic seed regenerates the identical switching sequence and decodes with zero overhead; a receiver that doesn't is driven toward random-guess error rates.

CodeRate 1/2 convolutional code, constraint length K=7, free distance ≥ 10
Polynomial selectionLogistic map (r = 3.99), quantized to a 2-bit index over 4 pre-chosen polynomial pairs
Decoding64-state hard-decision Viterbi decoder, synchronized to the same chaotic seed
Channel modelAWGN, BPSK modulation, simulated over 0–10 dB Eb/N0
HardwareVerilog RTL — Q8.16 fixed-point logistic map, combinational polynomial MUX, ACS-based Viterbi core
ToolchainMATLAB (simulation), Verilog (RTL), target FPGA (28nm-class fabric)
→ BER statistically equivalent to the fixed NASA (133,171) reference code across 0–10 dB Eb/N0 — about 10-4 at 6 dB and below 10-5 at 8 dB
→ Effective key space of roughly 252 under IEEE 754 double-precision seed arithmetic
→ A seed error as small as 10-12 decorrelates the chaotic sequence within about 40 iterations, driving an unsynchronized receiver's BER toward 0.5
→ Polynomial selection verified statistically uniform (chi-squared test, N = 106) with no detectable autocorrelation structure
→ Verilog RTL encoder/decoder estimated at under 5% LUT overhead versus a fixed Viterbi decoder, clocking above 100 MHz
→ Full reproducible MATLAB simulation suite and RTL published in the linked repository

3-Channel Digital Voltmeter (ADC0808 + 8051)

Embedded systems project — microcontroller programming
Complete

A 3-channel digital voltmeter built on the 8051 architecture. Converts analog voltage (0–5V) from potentiometers into a digital readout on a 16×2 LCD, with measured resolution under 20 mV and error below 2% across the test range.

MicrocontrollerAT89S52 (8051 architecture)
ADCADC0808, 8-channel multiplexer
Display16×2 HD44780 LCD, 4-bit mode
FirmwareEmbedded C, Keil µVision
SimulationProteus Design Suite
→ Average error under 2% across test voltages from 0.5V to 4.5V
→ Resolution of 19.5 mV per step (5V / 255)
→ Verified in Proteus with real-time, continuously updating display
→ Full project report with schematic, source code, and results documented

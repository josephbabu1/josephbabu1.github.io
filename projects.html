EV Go-Kart CAN Telemetry System

Formula Kart Racing Championship, Season 06 — March 2025 to March 2026
Complete

A complete CAN bus telemetry system for a competitive electric go-kart. Reads motor data — RPM, temperature, and voltage — from the VESC motor controller over CAN, and displays real-time metrics on an onboard LCD during practice and race runs.

MicrocontrollerESP32 with MCP2515 CAN controller
CAN protocol250 kbps, extended frames (29-bit IDs)
Display16×2 I2C LCD
Motor controllerVESC, custom DBC file
Speed calculationDerived from motor RPM and wheel circumference (27.94 cm diameter)
→ Tested and run on the actual go-kart during live practice sessions
→ Delivered as the telemetry subsystem for the team's Season 06 entry

Chaos-Driven Time-Varying Convolutional Codec

Research project — FPGA implementation, Verilog HDL
In Progress

A convolutional encoder (K=7) that varies its generator polynomials dynamically using a synchronized chaotic sequence, rather than the fixed generators used in conventional designs. The aim is to introduce lightweight physical-layer security while preserving standard error-correction performance.

EncodingRate 1/2 convolutional encoder, constraint length K=7
Polynomial selectionDynamic, driven by a logistic-map chaos generator
Decoding64-state Viterbi decoder, synchronized to the chaos generator
ImplementationSynthesizable Verilog HDL, target FPGA
ToolchainVivado, ModelSim, Xilinx FPGA
→ BER simulation underway against the fixed NASA (133,171) reference code
→ Chaos generator synchronization verified for correct decoding
→ Resource usage (LUTs, flip-flops, max clock frequency) being characterized
→ Full technical proposal completed, with block diagrams and architecture

3-Channel Digital Voltmeter (ADC0808 + 8051)

Embedded systems project — microcontroller programming
Complete

A 3-channel digital voltmeter built on the 8051 architecture. Converts analog voltage (0–5V) from potentiometers into a digital readout on a 16×2 LCD, with measured resolution under 20 mV and error below 2% across the test range.

MicrocontrollerAT89S52 (8051 architecture)
ADCADC0808, 8-channel multiplexer
Display16×2 HD44780 LCD, 4-bit mode
FirmwareEmbedded C, Keil µVision
SimulationProteus Design Suite
→ Average error under 2% across test voltages from 0.5V to 4.5V
→ Resolution of 19.5 mV per step (5V / 255)
→ Verified in Proteus with real-time, continuously updating display

FPGA-Based Crypto Accelerator (AES-128 + SHA-256)

Hardware security module — tamper response, ESP32 host bridge
In Progress

A dedicated FPGA hardware accelerator for AES-128 encryption/decryption and SHA-256 hashing, with a protected internal key register that cannot be read out once configured — only used internally. A physical tamper switch zeroizes the key instantly, demonstrating active tamper response. An ESP32 acts as a USB-to-SPI bridge so a host PC can issue encryption and HMAC commands over Python, while the secret key never leaves the FPGA, even if the host is compromised.

AES-128ECB and CBC modes, fully pipelined for throughput
SHA-256HMAC support for authentication
Key storageInternal register, write-only, locked after programming
Tamper responsePhysical switch triggers key zeroization and clears sensitive state
Host interfaceESP32 as SPI bridge, commands issued from a Python host script over USB
Target hardwareXilinx Artix-7 (or equivalent), Vivado toolchain
→ AES-128 core designed and simulated; SHA-256 module under implementation
→ Next: write-once key storage integration, tamper circuit on physical FPGA board, ESP32 and Python host firmware
→ Design notes and simulation results being documented as development continues
